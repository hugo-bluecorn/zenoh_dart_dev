## Overview

Phase 10 adds a declared querier — a long-lived entity for sending repeated queries to the same key expression. Structurally mirrors Phase 3's Publisher: declaration-time options (target, consolidation, timeout) are fixed at creation, while per-query options (payload, encoding) vary per `get()` call. Reuses the NativePort reply callback infrastructure from Phase 6's `Session.get()` and the matching listener pattern from Phase 3's Publisher.

### Key Design Decisions

- **Reuse reply callback:** `zd_querier_get` reuses `_zd_reply_callback` and `_zd_get_drop` from `zd_get`
- **Declaration vs per-query split:** target/consolidation/timeout at declare, payload/encoding per-get
- **Matching listener mirrors Publisher:** Same NativePort + StreamController\<bool\> pattern
- **Stream\<Reply\> return type:** Matches `Session.get()` pattern, completes on null sentinel

## Overview

Phase 12 is both a benchmark tooling phase and a vehicle for three new API features. Background subscriber provides fire-and-forget stream semantics matching zenoh-c's `z_declare_background_subscriber`. Publisher express mode adds the last missing `z_publisher_options_t` field (excluding unstable API). ZBytes `clone()` and `toBytes()` complete the read/copy API. The z_ping/z_pong CLIs implement a round-trip latency benchmark using all three features.

### Key Design Decisions

- **Background subscriber returns `Stream<Sample>`** — no handle, no close, lives until session closes
- **Publisher `is_express` uses -1 sentinel** — consistent with congestion_control/priority, preserves backward compatibility
- **Collapsed slice API** — `zd_bytes_to_buf` hides `z_bytes_to_slice`/`z_slice_data`/`z_slice_drop` internally
- **Slices 1-4 are independent** — O'Brien can implement in any order

## Plan Summary

Full bytes serialization/deserialization API enabling cross-language typed data exchange.
Three API layers matching zenoh-c:

1. **Convenience** -- `ZBytes.fromInt()` / `toInt()` for single-value round-trips
2. **Composite** -- `ZSerializer` / `ZDeserializer` for multi-value payloads
3. **Raw** -- `ZBytesWriter` / slice iterator for byte-level assembly and fragmented reading

**Scope:** +46 C shim functions (92->138), +3 Dart classes, +1 CLI example, ~61 tests (394->~455)

## Key Decisions

### No one-shot C shim functions for convenience methods
ZBytes convenience methods (`fromInt`/`toInt`, etc.) use `ZSerializer`/`ZDeserializer`
internally in Dart rather than adding dedicated C shim functions. This keeps the
C shim minimal and leverages the existing serialization infrastructure.

### Reader API deferred
The `ZBytesReader` (byte-level read from ZBytes) is not included. The slice iterator
(`ZBytes.slices`) provides fragmented read access. A sequential reader would require
additional C shim functions and has no immediate use case beyond what the
deserializer already provides.

### Test count expansion rationale
61 tests for 46 C shim functions reflects the serialization domain:
- Slices 2-3 can only verify "no error" (serializer output is opaque without deserializer)
- Slice 4 validates all round-trips (19 tests for 14 type methods + lifecycle + edge cases)
- Slices 5-6 test composite patterns and convenience wrappers at the Dart level (no new C shim)
- Slice 9 tests error paths that are critical for developer experience

### ze_deserializer_t is a copyable stack struct
Unlike most zenoh-c owned types, `ze_deserializer_t` has no drop function. The C shim
heap-allocates it (via sizeof + memcpy), and Dart frees with `calloc.free()`. This is
consistent with its value-type semantics in zenoh-c.

### String serialization uses ze_serializer_serialize_str
The C shim `zd_serializer_serialize_string` wraps `ze_serializer_serialize_str`
(null-terminated C string), NOT `ze_serializer_serialize_string` (which takes a
`z_moved_string_t`). This simplifies the Dart FFI -- pass a `Pointer<Utf8>` directly.
